  <name>Task 1: Refactor intent-classifier.ts to use NLP.js primary + Haiku fallback</name>
  <files>src/lib/ai/intent-classifier.ts</files>
  <action>
    Modify `classifyIntent()` in intent-classifier.ts:

    1. **Add import** at top: `import { getNlpClassifier } from './nlp-classifier';`

    2. **Remove `keywordFallback()` function entirely** -- its patterns are now in training-data.json.

    3. **Remove the confidence-override post-processing block** (the secondary safety net that checks for `result.confidence < 0.8` and overrides with regex). NLP.js handles these cases deterministically.

    4. **Restructure `classifyIntent()`:**
       ```
       a. Try NLP.js first:
          - const nlp = getNlpClassifier();
          - const nlpResult = await nlp.classify(message);
          - If nlpResult.confidence >= 0.5:
            - Extract params from entities + message text using an extractParams() helper
            - Return { intent: nlpResult.intent, confidence: nlpResult.confidence, params }
          - Log: '[Intent Classifier] NLP.js classified: {intent} ({confidence})'

       b. If NLP.js confidence < 0.5, fall through to existing Haiku API call:
          - Log: '[Intent Classifier] NLP.js low confidence ({confidence}), falling back to Haiku'
          - Keep existing Haiku fetch logic (KILO_BASE_URL, CLASSIFICATION_MODEL, etc.)
          - Keep the JSON parsing and validation of Haiku response
          - Return Haiku result as before
       ```

    5. **Create `extractParams()` helper function** that takes (intent, entities, message) and returns params:
       - For budget intents: extract amount via regex `/\b(\d[\d\s]*)\b/` from message, extract name from NLP.js entities or leftover text
       - For checklist/guest intents: extract title/name from entities or the message text
       - For query intents: determine filter param from message keywords (same logic as current keyword fallback had)
       - This is a best-effort extraction -- Sonnet will further refine params in the chat response. The key job of NLP.js is getting the intent right; params are supplementary.

    6. **Keep unchanged:** IntentResult interface, isActionIntent(), isQueryIntent(), isDemandSignal(), CLASSIFICATION_SYSTEM_PROMPT (kept for Haiku fallback), KILO_BASE_URL, CLASSIFICATION_MODEL constants.

    7. **Error handling:** If NLP.js throws (e.g., init failure), catch and fall back to Haiku. Log the error.
  </action>
