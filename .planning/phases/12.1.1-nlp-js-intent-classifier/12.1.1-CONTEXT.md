# Phase 12.1.1: NLP.js Intent Classifier - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace Haiku API classifier with locally-trained NLP.js model for deterministic, free, fast intent classification with Czech language support. Train on existing 50+ examples plus generated variants (~150-200 total). Keep Sonnet for chat responses only. Action-executor interface (IntentResult) stays the same -- drop-in replacement.

</domain>

<decisions>
## Implementation Decisions

### Fallback Strategy
- NLP.js is primary classifier, keyword fallback removed (absorbed into NLP.js training data)
- NLP.js confidence < 0.5 -> fallback to Haiku API (existing classifier as backup)
- NLP.js confidence >= 0.5 -> use NLP.js result, no Haiku call
- Action execution threshold stays at 0.6 (same as now)

### Training Data
- Source: existing 50+ examples from CLASSIFICATION_SYSTEM_PROMPT + 3-5 generated variants per intent (different formulations, typos, colloquial Czech)
- Target: ~150-200 training examples total
- Format: JSON file at src/lib/ai/training-data.json -- NLP.js native corpus format
- Versioned in git, editable without code changes
- New intents: add examples to JSON + redeploy. NLP.js retrains on server cold start.

### Entity Extraction
- NLP.js extracts intent + basic entities (name/title, amount as number)
- Date extraction left to parseCzechDate (existing)
- Tags/categories left to Sonnet context or NLP.js entity if simple enough
- Output: same IntentResult interface { intent, confidence, params } -- drop-in replacement for action-executor

### Claude's Discretion
- NLP.js package choice (@nlpjs/core vs node-nlp vs nlp.js monorepo)
- Exact NLP.js configuration (language, stemmer, tokenizer settings)
- How to handle NLP.js initialization (singleton, lazy load, module-level)
- Exact number of synthetic variants per intent
- Whether to pre-train and cache model or train on every cold start

</decisions>
