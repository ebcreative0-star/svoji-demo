# Phase 12.1.1 Plan 02: NLP.js Integration into Production Classifier Summary

**One-liner:** NLP.js replaces Haiku as primary intent classifier in intent-classifier.ts with Haiku retained as fallback for None/low-confidence results, removing keywordFallback and confidence-override post-processing.

## What Was Built

### intent-classifier.ts (refactored)

`classifyIntent()` now follows a two-stage pipeline:

1. **NLP.js primary** -- calls `getNlpClassifier().classify(message)`. If result is not "None" intent and confidence >= 0.5, extracts params via `extractParams()` and returns immediately. Zero network calls, < 50ms latency.

2. **Haiku fallback** -- fires when: NLP.js returns "None" intent (no training match), NLP.js confidence < 0.5, or NLP.js throws. Keeps the full existing Haiku fetch logic unchanged.

Removed:
- `keywordFallback()` function -- training-data.json now covers all 21 intents
- Confidence-override post-processing block -- NLP.js handles these cases deterministically
- `grep -c 'keywordFallback' intent-classifier.ts` returns 0

Added:
- `import { getNlpClassifier } from './nlp-classifier'`
- `extractParams()` helper: best-effort param extraction from NLP.js entities + regex for amounts, titles, query filters
- `isNoneIntent` guard: NLP.js returns `intent: "None"` with `score: 1` for unrecognized inputs -- explicit check prevents false high-confidence pass-through

### nlp-classifier.test.ts (extended)

Four new Plan 02 integration tests added after the six Plan 01 tests:
- **INTENT-05 no-fetch**: spies on `globalThis.fetch`, classifies "pridej fotografa do checklistu", asserts fetch NOT called
- **None intent fallback**: classifies nonsense input, asserts `intent === "None"` (confirming fallback path trigger)
- **Param extraction**: classifies "zaplatil jsem fotografa 25000", asserts `params.amount === 25000` and `params.name` contains "fotograf"
- **keywordFallback removed**: imports intent-classifier module, asserts `typeof keywordFallback === 'undefined'`

All 10 tests pass.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] NLP.js returns intent="None" with score=1 for unrecognized inputs**
- **Found during:** Task 2 (test execution)
- **Issue:** When no training utterance matches, NLP.js does not return low confidence -- it returns `{ intent: "None", score: 1 }`. Without a guard, the primary path would accept "None" as a high-confidence result and never fall through to Haiku.
- **Fix:** Added `isNoneIntent` check (`intent === 'None' || intent === 'none'`) before the confidence threshold check. None intent always routes to Haiku fallback.
- **Files modified:** `src/lib/ai/intent-classifier.ts`, `src/lib/ai/__tests__/nlp-classifier.test.ts`
