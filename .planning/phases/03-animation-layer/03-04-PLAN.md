**Fix scroll-behavior conflict (Gap 2):**

In `src/app/globals.css`, line 54, remove `scroll-behavior: smooth;` from the `html` rule.

Change:
```css
html {
  scroll-behavior: smooth;
  -webkit-text-size-adjust: 100%;
  overflow-x: hidden;
}
```

To:
```css
html {
  -webkit-text-size-adjust: 100%;
  overflow-x: hidden;
}
```

Rationale: Lenis handles smooth scroll via JavaScript. When reduced-motion is enabled, LenisProvider skips Lenis and native scroll should be instant (no animation). Keeping `scroll-behavior: smooth` in CSS means reduced-motion users still get animated scroll -- violating accessibility expectations.
