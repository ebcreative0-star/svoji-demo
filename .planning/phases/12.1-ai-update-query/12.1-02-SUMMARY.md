## Decisions Made
- Query intents reuse the same executeAction() pipeline as mutations but distinguish themselves via `data.type = 'query'` in the returned ActionResult -- allows the same STEP 3 to branch correctly
- DATA PRO ODPOVED prompt injection is separate from AKCE PROVEDENA to give the AI clear semantic signal: "you're answering a question, not confirming a mutation"
- statusOverview uses parallel Promise.all() across 4 tables to keep latency low
- Filters applied in JS not SQL for simplicity -- per-couple data sets are small (typically <200 items)
