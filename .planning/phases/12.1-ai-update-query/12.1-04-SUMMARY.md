## What Was Built

### Task 1: Czech Word Numbers + Date Error Handling

`src/lib/date-utils.ts`:
- Added `CZECH_WORD_NUMBERS` map covering jeden/dva/tri/ctyri/pet/sest/sedm/osm/devet/deset
- New regex handlers for `za [word] dni` and `za [word] tydny` patterns
- NFD normalization strips diacritics before map lookup (handles 'dva', 'dve', etc.)
- Existing digit-based regexes unchanged

`src/lib/ai/action-executor.ts`:
- `updateChecklistItem`: replaced silent fallback `parsed ?? cleanUpdates.due_date` with explicit error return when `parseCzechDate` returns null
- Error message is user-friendly Czech text with format examples

### Task 2: Query De-truncation + Formatting Hardening

`src/lib/ai/action-executor.ts`:
- Removed `.slice(0, 15)` from `queryChecklist`, `queryBudget`, `queryGuests`
- `queryChecklist` now groups items by category when >30 items (prevents context bloat)

`src/app/api/chat/route.ts`:
- System prompt: `Nepouzivej emoji prehrane` -> `NIKDY nepouzivej emoji. Ani jediny.`
- System prompt: weak bold rule replaced with `NIKDY nepouzivej markdown formatovani`
- DATA PRO ODPOVED injection: stronger formatting instruction
- Post-processing strip added after AI response: removes `**bold**`, `*italic*`, `# headings`, and emoji ranges via Unicode regex

`src/lib/kilo.ts`:
- Temperature: 1.0 -> 0.7

### Task 3: Classifier Keyword Fallback

`src/lib/ai/intent-classifier.ts`:
- New `keywordFallback()` function with 6 pattern groups: guest_query, checklist_query, budget_query, status_overview, checklist_update, budget_mark_paid
- Called before Haiku API -- returns immediately on match, skipping API latency
- Secondary confidence-override layer after Haiku: if confidence <0.8 and intent is advice_request/small_talk, regex checks override to correct intent + params
- Addresses UAT test failures for "kolik hostu potvrdilo" and "pridej datum k fotografovi"
