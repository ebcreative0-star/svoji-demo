Create `src/components/dashboard/SearchModal.tsx` as a 'use client' component:

1. Define SearchItem interface:
```typescript
interface SearchItem {
  id: string;
  title: string;
  domain: 'checklist' | 'budget' | 'guests';
  subtitle?: string;  // e.g. category, amount, group
  tags?: string[];
  href: string;  // e.g. '/checklist?highlight=uuid'
}
```

2. Props: `{ items: SearchItem[], open: boolean, onOpenChange: (open: boolean) => void }`

3. Search logic (client-side filter, no API):
   - `filterItems(query, items)`: lowercase includes on title + tags + subtitle
   - Group results by domain: CHECKLIST / ROZPOCET / HOSTE sections
   - Show max 5 results per domain (15 total max)
   - Empty query = no results shown (not all items)

4. UI:
   - Overlay backdrop with AnimatePresence fade
   - Centered modal card (max-w-lg) with motion.div scale animation
   - Search input at top with Search icon, autofocus
   - Results grouped under domain headers (bold section names)
   - Each result: title + subtitle + tags as small pills
   - Click result: call `router.push(item.href)` then `onOpenChange(false)`
   - Keyboard: Escape closes, Enter selects first result, Arrow keys navigate (optional -- keep simple if complex)
   - Show "Zadejte hledany vyraz..." when empty, "Nic nenalezeno" when no matches

5. Cmd+K / Ctrl+K binding: useEffect keydown listener inside the modal component.
   - On (metaKey || ctrlKey) + 'k': preventDefault, call onOpenChange(true)
   - On Escape: call onOpenChange(false)

Modify `src/app/(dashboard)/layout.tsx`:

1. Keep it as a server component. After the existing couple fetch, add 3 lightweight queries:
```typescript
const [{ data: checklistItems }, { data: budgetItems }, { data: guestItems }] = await Promise.all([

  supabase.from('budget_items').select('id, name, category, estimated_cost, tags').eq('couple_id', user.id),
```

2. Transform into SearchItem[] array:

   - Guests: { id, title: name, domain: 'guests', subtitle: group_name, tags, href: '/guests?highlight='+id }

3. Create a new `DashboardClientShell` wrapper component ('use client') that:
   - Receives `searchItems: SearchItem[]` and `children`
   - Manages `const [searchOpen, setSearchOpen] = useState(false)`
   - Renders `<SearchModal items={searchItems} open={searchOpen} onOpenChange={setSearchOpen} />`
   - Renders `{children}`
   - This can live in the same file as SearchModal or be a separate small component

4. Wrap `<main>` content in the layout with `DashboardClientShell`:

```tsx
  <main className="pt-16 pb-20 md:pb-0">{children}</main>
```

   - BUT DashboardNav is rendered in the server layout, not inside DashboardClientShell...

   - DashboardClientShell renders: DashboardNav (with onSearchOpen) + SearchModal + children
