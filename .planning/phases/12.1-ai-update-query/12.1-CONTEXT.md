# Phase 12.1: AI Update & Query Intents - Context

**Gathered:** 2026-03-16
**Status:** Ready for planning

<domain>
## Phase Boundary

AI chat can update existing items (not just create), answer data queries about checklist/budget/guests, provide cross-domain status overviews, understand tags/categories when creating items, and global search across all domains with tag matching.

</domain>

<decisions>
## Implementation Decisions

### Query Response Format
- Stručný přehled: souhrn + top položky, ne kompletní výpis (funguje i pro 50+ položek)
- Inline text formát, ne emoji karty: "Celkový rozpočet: 450 000 Kč, zaplaceno 180 000 Kč (40%)"
- status_overview ("jak jsem na tom?") vrací cross-domain souhrn všech tří domén najednou v jedné odpovědi

### Update Disambiguation
- Classifier rozliší update vs create: nový intent `checklist_update` v classifieru, Haiku rozpozná z kontextu ("přidej datum K položce" = update, "přidej položku" = create)
- Při více matchích se AI zeptá: "Našel jsem 2 položky: 1) X 2) Y. Kterou myslíš?"
- budget_mark_paid: pokud položka neexistuje, vytvořit novou + označit jako zaplacenou (ne chyba)
- budget_mark_paid: nastavit actual_cost na zmíněnou částku, zachovat estimated_cost, paid=true

### Global Search UX
- Cmd+K modal (na mobilu ikona lupy v headeru)
- Výsledky ze všech domén (checklist, rozpočet, hosté) seskupené podle domény
- Hledá v názvech/title + v tazích (SEARCH-01)
- Klik na výsledek: navigace na příslušný view + highlight položky (krátký flash efekt)
- Client-side search: filtruje již načtená data v paměti, žádný extra API call

### Data Context pro AI
- Fetch při query intentu: classifier rozpozná query -> action-executor načte data z DB -> vloží do system promptu
- Všechny položky (title, completed, due_date, tags, amounts) v textu do system promptu -- svatba má max desítky položek, vejde se
- Query intenty procházejí přes action-executor (stejný flow jako mutace, jen SELECT)
- Žádný markdown bold v AI odpovědích: explicitní pravidlo v system promptu "Nepoužívej **bold** markup. Piš čistý text s odrážkami a čísly."

### Claude's Discretion
- Přesný formát dat v system promptu (JSON vs plain text)
- Debounce/delay pro client-side search
- Flash highlight implementace (Framer Motion vs CSS animation)
- Řazení výsledků v search modalu
- Cmd+K shortcut binding implementace

</decisions>

## Existing Code Insights

### Reusable Assets
- `intent-classifier.ts`: Haiku classifier s 15+ intenty, confidence scoring, param extraction -- rozšířit o checklist_update, budget_mark_paid, query intenty
- `action-executor.ts`: CRUD switch/case, ilike partial match -- rozšířit o query handlers (SELECT only)
- `isActionIntent()`: seznam action intentů -- rozšířit o nové intenty
- `buildSystemPrompt()` v route.ts: dynamický prompt -- rozšířit o data injection pro query responses
- Tags jako `text[]` na všech tabulkách (migration 009 z Phase 12)
- `normalizeBudgetCategory()`: Czech->English category mapping -- již existuje

### Established Patterns
- Classifier -> action-executor -> system prompt injection -> Sonnet response (pipeline z Phase 8/11.1)
- ilike `%term%` pro partial match hledání položek
- Fire-and-forget pro logging, synchronní pro akce
- Haiku temp 0.1 pro classification, 0.6 confidence threshold

### Integration Points
- `/api/chat/route.ts`: orchestrace -- přidat query data injection do system promptu
- `intent-classifier.ts`: přidat nové intenty + příklady
- `action-executor.ts`: přidat query handlery + checklist_update + budget_mark_paid
- Dashboard layout: přidat Cmd+K search modal (nový komponent)
- Dashboard views: přijmout highlight prop pro flash efekt po search navigaci
