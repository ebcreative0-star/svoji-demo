<objective>
Add query intents (checklist_query, budget_query, guest_query, status_overview) to the AI pipeline so the chatbot can answer questions about the user's actual data from the database.

Purpose: Users currently cannot ask "what's in my checklist?" or "how much have I spent?" -- the AI has no access to their data. This plan makes query intents flow through the same classifier -> action-executor pipeline but return data for system prompt injection instead of mutating the DB.

Output: 4 new query intents in classifier, 4 query handlers in action-executor returning formatted context strings, route.ts injects query data into system prompt.
</objective>

   - Format as plain text context: "Checklist: X nesplenich, Y hotovych.\n" + up to 15 pending items as "- title (do date) [tags]"
   - Return { success: true, message: contextString, data: { type: 'query', context: contextString } }

2. Add `queryBudget` handler:
   - SELECT name, category, estimated_cost, actual_cost, paid, tags FROM budget_items WHERE couple_id
   - Also SELECT budget_total FROM couples WHERE id = coupleId
   - Compute: totalEstimated = SUM(estimated_cost), totalPaid = SUM(actual_cost WHERE paid=true), unpaidCount, paidCount
   - If budget_total exists: remaining = budget_total - totalPaid
   - Apply filter: 'unpaid' = show unpaid items, 'paid' = show paid items, 'all' = show summary + top items
   - Format: "Rozpocet: celkem X Kc odhadovano, Y Kc zaplaceno (Z%). Zbyva: W Kc.\n" + item list (up to 15)
   - Return with data.type = 'query'

3. Add `queryGuests` handler:
   - SELECT name, group_name, rsvp_status, plus_one, tags FROM guests WHERE couple_id
   - Group by rsvp_status in JS: confirmed count, pending count, declined count, total
   - Apply filter to item list
   - Format: "Hoste: X celkem, Y potvrzenych, Z cekajicich, W odmitnutych.\n" + item list (up to 15)
   - Return with data.type = 'query'

4. Add `statusOverview` handler:
   - Execute 3 SELECTs (checklist, budget, guests) for couple_id

     - Budget: estimated total, paid total, remaining (vs budget_total if set)
     - Guests: total, confirmed, pending
   - Cap at ~500 chars total -- summary only, no item lists
   - Return with data.type = 'query'
