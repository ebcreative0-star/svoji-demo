# Plan 08-02: Intent Classification with Actions

## Goal
Classify user messages into intents and execute actions (checklist/budget/guest CRUD) based on classification.

## Context
- AI should be able to manipulate user's data through natural conversation
- Intent classification runs async (fire-and-forget for logging, sync for actions)
- Actions must be executed BEFORE returning AI response

## Intent Categories

### Action Intents (modify data)
| Intent | Description | Action |
|--------|-------------|--------|
| `checklist_add` | Add item to checklist | Insert into `checklist_items` |
| `checklist_complete` | Mark item done | Update `is_completed = true` |
| `checklist_remove` | Delete item | Delete from `checklist_items` |
| `budget_add` | Add expense | Insert into `budget_items` |
| `budget_update` | Update amount | Update `amount` field |
| `budget_remove` | Delete expense | Delete from `budget_items` |
| `guest_add` | Add guest | Insert into `guests` |
| `guest_update` | Change RSVP/details | Update guest record |
| `guest_remove` | Remove guest | Delete from `guests` |

### Informational Intents (log only)
| Intent | Description |
|--------|-------------|
| `vendor_search` | Looking for vendor (fotograf, catering, etc.) |
| `advice_request` | General wedding advice |
| `small_talk` | Off-topic conversation |

## Tasks

### 1. Create intent classifier
**File:** `src/lib/ai/intent-classifier.ts`
- Use Kilo Gateway with cheaper/faster model for classification
- Input: user message + conversation context
- Output: `{ intent: string, confidence: number, params: object }`
- Params contain extracted data (e.g., item name, amount, guest name)

### 2. Create action executor
**File:** `src/lib/ai/action-executor.ts`
- Switch on intent type
- Execute appropriate Supabase mutation
- Return action result for AI to acknowledge

### 3. Create demand signal logger
**File:** `src/lib/ai/demand-logger.ts`
- For `vendor_search` intents
- Extract: category, region, budget_hint, urgency
- Insert into `demand_signals` table (create in next plan)

### 4. Integrate into chat route
**File:** `src/app/api/chat/route.ts`
- Before calling main AI:
  1. Classify intent
  2. If action intent: execute action
  3. Add action result to AI context
- After AI response:
  1. If vendor_search: log demand signal (async)

### 5. Update AI system prompt
- Add instructions about confirming actions
- "Kdyz uzivatel chce pridat/upravit/smazat neco, potvrď akci"
