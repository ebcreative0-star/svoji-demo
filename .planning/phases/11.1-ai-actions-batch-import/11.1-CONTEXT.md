# Phase 11.1: AI Actions & Batch Import - Context

**Gathered:** 2026-03-14
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix AI chat tool execution reliability (AI has tools but doesn't use them properly), add batch import capabilities for checklist/budget/guests, implement notes migration (paste text from mobile notes, AI categorizes), fix chat persistence, and investigate phantom budget item. The AI chat system already has intent classification and action executor - this phase fixes reliability and extends capabilities.

</domain>

<decisions>
## Implementation Decisions

### Notes Migration
- Chat-only entry point (no separate import page or UI)
- AI detects long multi-line pasted text as notes migration intent
- Parse & execute immediately - AI categorizes items and adds them without asking for confirmation
- Smart inference for ambiguous items: numbers indicate budget, action verbs indicate checklist, names indicate guests. Only ask if truly ambiguous
- Show summary table after import: breakdown by category (e.g. "8 polozek pridano: 5x checklist, 2x rozpocet, 1x host")

### Welcome Message
- Dynamic welcome message that adapts based on couple's data state
- Appears immediately on first chat open
- New users: capability overview + import hints + nudge to paste existing wedding plan
- Returning users with empty checklist: nudge to paste notes
- Users with data: contextual tips based on what they have/missing
- Include suggested actions (what AI can do: add to checklist, budget, guests, get advice)

### Batch Operations
- Execute immediately without confirmation, regardless of batch size
- Show summary table with category breakdown after batch execution
- User can undo individual items in their respective dashboard views (checklist/budget/guests)

### Chat Persistence
- Load full message history from DB when chat opens (currently saves to DB but doesn't reload on tab switch)
- Add date headers between messages from different days ("Dnes", "Vcera", "12. brezna")
- Fix the tab-switch reload bug so messages persist across navigation

### Phantom Budget Item
- Likely from AI testing - orphaned items in DB not showing in UI
- Investigate root cause and clean up existing bad data

### Claude's Discretion
- Root cause fix + prevention approach for phantom items
- Exact welcome message copy and suggested prompts
- Loading/empty states for chat history
- Error handling for failed batch operations

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `intent-classifier.ts`: Full intent system with 12+ intent types, confidence scoring, param extraction
- `action-executor.ts`: CRUD for checklist, budget, guests - includes batch guest add (guest_add_multi)
- `ChatInterface.tsx`: Optimistic updates, rate limiting UI, suggested questions on empty state
- `buildSystemPrompt()` in `/api/chat/route.ts`: Dynamic prompt with couple context

### Established Patterns
- Kilo Gateway: Haiku for classification (temp 0.1), Sonnet for responses
- Confidence > 0.6 threshold for action execution
- Case-insensitive ilike() matching for item lookup
- Fire-and-forget logging (engagement, demand signals)
- Czech language throughout AI responses

### Integration Points
- `/api/chat/route.ts`: Main orchestration (intent -> action -> response)
- `chat_messages` table: Already persists messages, loads last 50 on page open
- ChatInterface loads messages in useEffect but may not re-fetch on tab return
- `buildSystemPrompt()`: Where welcome message logic and capability hints should go
