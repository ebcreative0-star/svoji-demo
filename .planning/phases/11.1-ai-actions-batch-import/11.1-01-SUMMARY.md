## Accomplishments

- Classifier now reliably triggers budget_add for any message containing an amount + item name (5 new examples + explicit PRAVIDLA rule)
- Classifier now reliably triggers budget_remove for delete/cancel/remove phrasings (3 new examples + PRAVIDLA rule)
- AI-provided categories normalized via CATEGORY_MAP before DB insert -- items with "fotograf" become category "photo", etc.
- BudgetView catch-all merges any items with unknown categories into "Ostatni" group, eliminating phantom totals
- Amount validation fixed: `amount == null` instead of `!amount` so zero-cost items are accepted

## Decisions Made

- BudgetView catch-all retained as defense-in-depth: covers legacy items and any future edge cases where normalization is bypassed
- `== null` check instead of `!amount` to correctly allow zero-value budget items
