<objective>
Fix Google OAuth onboarding data loss by persisting onboarding data in a cookie before the OAuth redirect.

Purpose: Supabase OAuth strips custom query params from the redirectTo URL. The onboarding data encoded as ?onboarding=base64 never reaches the callback. This causes new Google users to loop back to /onboarding instead of landing on /chat.
Output: Updated register page and auth callback that use a cookie to bridge onboarding data across the OAuth redirect.
</objective>

```typescript
// Try cookie fallback for onboarding data (Supabase strips query params from OAuth redirectTo)
let onboardingRaw = onboardingParam;
if (!onboardingRaw) {
```
