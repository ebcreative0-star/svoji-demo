# Plan 06-05: UAT Gap Closure

## Context

UAT found 2 gaps (from 06-UAT.md):
1. **Test 2 (cosmetic):** Footer bg is black (`bg-[var(--color-text)]`), clashes with cream site palette. Text too close to edges (px-4).
2. **Test 4 (major):** "Web pro hosty" link missing in dashboard nav. DashboardNav has the slug conditional render, but layout.tsx never queries `wedding_websites` table and never passes the slug prop.

## Tasks

### Task 1: Fix SaasFooter background and padding

**File:** `src/components/ui/SaasFooter.tsx`

**Changes:**

1. Line 8: Replace `bg-[var(--color-text)]` with `bg-[var(--color-primary-dark)]`
   - `--color-primary-dark` is `oklch(45% 0.038 55)` -- a warm dark brown that fits the cream/warm palette
   - This keeps white text legible against a dark background without the jarring pure black

2. Line 9: Replace `px-4` with `px-6 sm:px-8`
   - Gives column text more breathing room from container edges
   - Responsive: 1.5rem mobile, 2rem on sm+

**Verification:**
- `grep -n 'color-text' src/components/ui/SaasFooter.tsx` returns zero matches (no black bg reference)
- `grep -n 'px-6' src/components/ui/SaasFooter.tsx` returns the container line

### Task 2: Pass slug to DashboardNav

**File:** `src/app/(dashboard)/layout.tsx`

**Changes:**

1. In the authenticated (non-demo) path, after the couples query (line 35-38), add a second query:
   ```typescript
   const { data: website } = await supabase
     .from('wedding_websites')
     .select('slug')
     .eq('couple_id', user.id)
     .single();
   ```

2. Pass `slug` to DashboardNav in the authenticated return (line 47-49):
   ```tsx
   <DashboardNav
     partner1={couple.partner1_name}
     partner2={couple.partner2_name}
     slug={website?.slug}
   />
   ```

3. In the demo mode path (line 15-18), also pass a demo slug so the link renders in demo mode too:
   ```tsx
   <DashboardNav
     partner1={DEMO_COUPLE.partner1_name}
     partner2={DEMO_COUPLE.partner2_name}
     slug="demo"
   />
   ```

**Verification:**
- `grep -n 'wedding_websites' src/app/(dashboard)/layout.tsx` returns the query line
- `grep -n 'slug' src/app/(dashboard)/layout.tsx` returns prop passing lines
- Visit dashboard on desktop: "Web pro hosty" link with Globe icon is visible in the top nav bar
