Two changes in this file:

**Change A -- Refactor the AKCE A DATA section in buildSystemPrompt() (lines 90-93).**

Replace the current block:
```
AKCE A DATA:
- Kdyz uzivatel chce pridat/upravit/smazat polozku (checklist, rozpocet, host), system automaticky provede akci
- Tvuj ukol je potvrdit akci a rict co bylo udelano
- Nezminuj technicky proces, jen vysledek ("Pridal jsem...", "Oznacil jsem...", "Aktualizoval jsem...")
- Pokud akce selhala, omluvni se a navrhni rucni postup
```

With a NEUTRAL version that does NOT instruct the AI to confirm actions:
```
AKCE A DATA:
- System muze automaticky provadet akce (pridavani/upravy/mazani polozek) na zaklade tvych zprav
- Pokud system provede akci, dostanes informaci o vysledku -- pak potvrdi co bylo udelano
- Pokud NEDOSTANES informaci o provedene akci, NIKDY nepotvrzuj ze jsi neco pridal/zmenil/smazal
- Nezminuj technicky proces, jen vysledek
```

The key difference: the old prompt unconditionally said "Tvuj ukol je potvrdit akci" which primed Sonnet to always confirm. The new version conditions confirmation on receiving an action result. This makes the three-way branching (lines 203-216) actually effective -- the base no longer contradicts the ZADNA AKCE guard.

**Change B -- Lower confidence threshold from 0.7 to 0.6 (line 189).**

Change:
```typescript
if (isActionIntent(intentResult.intent) && intentResult.confidence > 0.7) {
```
To:
```typescript
if (isActionIntent(intentResult.intent) && intentResult.confidence > 0.6) {
```

This prevents valid classifications at 0.61-0.69 from being silently dropped. The few-shot examples target 0.95, but real colloquial input may score 0.65-0.75. Combined with the three-way branching, a misfire at 0.61 will still produce an action attempt with an honest result rather than a silent skip.

**Change C -- Add diagnostic logging before the confidence check (after line 188).**

Add a console.log that prints the full intentResult object right before the confidence gate so runtime behavior is observable:

```typescript
console.log('[AI Pipeline] Intent result:', JSON.stringify(intentResult), '| Threshold: 0.6 | Will execute:', isActionIntent(intentResult.intent) && intentResult.confidence > 0.6);
```

This replaces the existing `console.log('Intent classification:', intentResult);` on line 185 with a more detailed version that also logs whether the action will fire.
